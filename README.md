# Quiz Battle

Веб-приложение для проведения квизов в формате «баттла»: пользователь последовательно выбирает победителя в паре карточек, а система формирует итоговый рейтинг и сохраняет статистику по элементам викторины.

## Состав команды и роли

### Разработчик 1 — Backend / База данных
Отвечал за:
- проектирование серверной части на Flask;
- описание моделей базы данных;
- настройку SQLAlchemy и Flask-Migrate;
- реализацию REST API;
- миграции БД и начальную структуру данных;
- написание backend-тестов.

### Разработчик 2 — Frontend / DevOps
Отвечал за:
- реализацию клиентской части на React + TypeScript;
- настройку маршрутизации и взаимодействия с API;
- сборку frontend-приложения;
- контейнеризацию frontend и backend;
- конфигурацию `docker-compose`;
- настройку reverse proxy через Nginx.

## Основной функционал

- просмотр списка доступных квизов;
- запуск квиза по выбранной теме;
- голосование между двумя вариантами;
- определение победителя квиза;
- отображение итогового рейтинга;
- хранение статистики побед и поражений;
- работа приложения через контейнеры Docker.

## Структура backend

В серверной части реализованы:
- фабрика приложения `create_app()`;
- модели `User`, `Quiz`, `QuizItem`, `BattleRound`;
- API-маршруты:
  - `GET /api/quizzes` — список квизов;
  - `GET /api/battle/<quiz_id>` — получить пару для голосования;
  - `POST /api/vote` — отправить голос;
  - `GET /api/winner/<quiz_id>` — получить победителя;
  - `GET /api/quiz_results/<quiz_id>` — получить итоговый рейтинг.

Также присутствуют миграции Alembic и тесты для моделей, роутов и фикстур.

## Структура frontend

В клиентской части реализованы:

- переходы между экранами через React Router;
- вызовы API в `api.ts`;
- компонентная структура (`QuizList`, `QuizGame`).

## Контейнеризация и запуск

В проекте предусмотрены 4 сервиса:
- `backend` — API-сервис;
- `frontend` — собранный клиент;
- `db` — PostgreSQL;
- `nginx` — единая точка входа.

### Запуск

```bash
docker compose up --build
```

После запуска:
- frontend доступен через Nginx;
- backend проксируется через `/api/`;
- база данных доступна внутри docker-сети.

## Nginx

Nginx используется как reverse proxy:
- запросы к `/` направляются на frontend;
- запросы к `/api/` направляются на backend.

Это позволяет использовать единую точку входа для клиентской и серверной частей приложения.

## Тестирование

Также настроен workflow GitHub Actions для автоматического запуска backend-тестов.
